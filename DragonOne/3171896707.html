Android - Kitsunebi 使用教程

本文最后更新于：December 3, 2021 pm

Kitsunebi 是一个基于 V2Ray 核心的移动平台应用。它可以创建基于 VMess 或者 Shadowsocks (SS) 的 VPN 连接。Kitsunebi 支持导入和导出与 V2Ray 兼容的 JSON 配置。

由于使用 V2Ray 核心，Kitsunebi 几乎支持 V2Ray 的所有功能，比如 Mux 和 mKCP 。
安卓版本较为稳定易用，可以作为一个备用选项使用。

目录

暗流域：进入

主要特点：

  • 支持单独订阅神机规则
  • 对 V2Rray 支持程度高
  • 支持的订阅格式：SIP002 URIs / VMess

系统环境

Android 9, Color OS V6.0
Kitsunebi 1.8.0

此文档适用于：N3RO
N3RO 机场用户需要将 SSD 订阅自行转换为 SIP002 URIs 订阅，详情请进群咨询，
文档中的某些内容可能随软件版本迭代而失效。

  • 也可以直接在 Play Store 搜索 Kitsunebi
  • 或点击这里下载。

快速上手

添加服务器 (节点)

无订阅情况下：

支持三种导入方式：

  • 手动添加
  • 扫码添加
  • 从 URI 导入

有订阅情况下（推荐）

请在机场官网复制订阅地址。N3RO 用户请复制 SIP002 URIs 地址。
点击软件右上角的三个点 ，然后点击订阅。点击加号后粘贴你的订阅链接。

节点备注名称可以随便填写，本教程中写作“测试”。您可以自由备注您的节点信息。

若干秒后软件首页会出现节点信息。

延迟测试

点击右上角的环形箭头按钮，可进行延迟测试。节点延迟将显示在各个节点卡片上。

启动代理（连接节点）

此时您可以选择一个延迟较低的节点，然后点击右下角按钮，开始科学上网。

第一次启动时会询问是否创建 VPN 连接，请点击允许。

添加规则集订阅

规则用于帮助软件判断设备的访问请求是否需要被代理，避免出现国内服务速度变慢等问题，也可节省您的套餐流量。

请在断开节点连接的状态下添加规则集订阅。

请点击软件首页点击右上角三个点，再点击 规则集 → 再点击右上角三个点后选择新建 → 点击卡片上的铅笔图标

→ 然后点击右上角加号，选择“设置更新URL”，并将下边的规则网址粘贴至弹出的窗口，点击确定。

随后可以点击卡片右下角的刷新按钮更新订阅。

规则订阅地址：

  • 标准版： https://raw.githubusercontent.com/ConnersHua/Profiles/master/Kitsunebi/Basic.conf
  • 专业版 (去广告) ： https://raw.githubusercontent.com/ConnersHua/Profiles/master/Kitsunebi/Pro.conf
  • 回国版： https://raw.githubusercontent.com/ConnersHua/Profiles/master/Kitsunebi/China.conf

此章节内的操作非必要操作，请视个人需求添加。

添加节点组

安卓版 Kitsunebi 支持负载均衡。

有两种实现方式：

  • Latency：根据节点延迟值来选择延迟最低的节点
  • Random ：随机选择节点使用

使用方法：

请点击软件首页右上角三个点 → 点击添加节点组 → 选择要添加的节点 → 设置节点备注和策略。

建议使用 Latency 策略。然后在首页点击节点组卡片。打开开关即可使用，节点会自动切换。

分应用代理（按需设置，如无必要可以不设置）

Kitsunebi 的分应用代理功能比 SSR 小飞机 的更高级，可以和分流规则结合使用。

设置方法：

点击软件右上角三个点 → 点击 设置 → 分应用代理 → 开启分应用代理 → 选择分应用代理模式 ，然后按需添加所需要代理的软件。
